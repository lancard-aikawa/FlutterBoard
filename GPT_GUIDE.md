# CLAUDE.md（トークン消費最適化ガイド）

## 🎯 目的

Claude利用時のトークン消費を最小化し、効率的に開発・レビューを行う

---

## 🔥 基本原則

### 1. 全部読ませない

* 大きなファイルを丸ごと読ませない
* 必要な箇所だけを対象にする

---

### 2. 同じ内容を何度も読ませない

* 一度読んだ内容は再読しない
* サブエージェントでの再読を避ける

---

### 3. コンテキストを肥大化させない

* 長い会話を続けない
* 用途ごとにスレッドを分ける

---

## 📦 ファイル構成ルール（最重要）

### ❌ NG

* 1ファイルに数千行（例：app.js 4000行）

### ✅ OK

* 1ファイル 500〜800行以内
* 機能単位で分割

```
/src
  ├── main.js
  ├── ui.js
  ├── api.js
  ├── state.js
```

---

## 🔍 読み込み戦略

### ❌ NG

* ファイル全体を読む
* 分割して何度も読む

### ✅ OK

* 検索 → 必要箇所のみ読む

例：

```
"handleSubmit が定義されている箇所だけ見て"
```

---

## ✏️ 編集ルール

### ❌ NG

* 編集後に全体を再読する
* 「今の状態どう？」と確認する

### ✅ OK

* 差分ベースで確認
* 編集結果は再読不要（状態は保持されている前提）

---

## 🤖 サブエージェント利用

### ❌ NG

* コードレビューを丸投げ
* 大量ファイルを再読させる

### ✅ OK

* 原則使用しない
* 必要なら対象範囲を極小化する

---

## 🛠 スクリプト実行

### ❌ NG

* ファイル書き出し → 読み込み → 実行

### ✅ OK

* ワンライナーで実行

```
node -e "処理"
```

```
sed -i
```

---

## 💬 会話運用ルール

### ❌ NG

* 1スレで全部やる

  * 設計
  * 実装
  * レビュー
  * デバッグ

### ✅ OK

* スレッド分離

```
・設計用
・実装用
・バグ調査用
```

---

## 🚀 推奨ワークフロー

1. Grep等で対象箇所を特定
2. 必要部分のみClaudeに渡す
3. 修正は差分ベースで依頼
4. 再読しない

---

## 📘 上級テクニック

### 要約キャッシュを持つ

`ARCHITECTURE.md` を作成：

```
・各ファイルの役割
・主要関数一覧
```

👉 毎回コード全体を読ませる必要がなくなる

---

### 関数単位で依頼する

```
この関数だけレビューして
```

👉 トークン消費を大幅削減

---

## 🧠 まとめ

最も重要なポイント：

* ファイルを分割する
* 必要な箇所だけ読む
* 同じ内容を繰り返し読ませない
* 長い会話を続けない

---

> 「全部読むな、何度も読むな、長く持つな」
