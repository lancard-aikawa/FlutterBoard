applicationId の変更

⚠️ 警告

問題

applicationId が Flutter テンプレートの既定値 com.example.* のままです。 この ID は Google Play Store でアプリを一意に識別するため、リリース前に必ず変更してください。 一度 Play Store に公開した後は変更できません。

変更場所

Groovy DSL（android/app/build.gradle）:

android {
    defaultConfig {
        applicationId "com.yourcompany.yourapp"  // ← ここを変更
        ...
    }
}

Kotlin DSL（android/app/build.gradle.kts）:

android {
    defaultConfig {
        applicationId = "com.yourcompany.yourapp"  // ← ここを変更
        ...
    }
}

命名規則

  • 逆ドメイン記法が推奨: com.yourcompany.yourapp
  • 小文字・数字・アンダースコア・ドットのみ使用可
  • セグメントは英字で始まること（例: com.1app は不可）
  • 既に Play Store に登録されている ID は使用不可

iOS の Bundle ID も合わせて変更する

Xcode で ios/Runner.xcworkspace を開き、Runner ターゲットの Bundle Identifier を同じ ID に変更してください。

open ios/Runner.xcworkspace
注意: applicationId を変更すると、既存のインストール済みアプリとは別アプリとして扱われます。 テスト端末の既存アプリを一度アンインストールしてから再インストールしてください。
