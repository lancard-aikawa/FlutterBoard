print / debugPrint の除去

⚠️ 警告

問題

lib/ 配下の Dart コードに print() または debugPrint() が残っています。 リリースビルドに含まれるとコンソールに不要なログが出力され、機密情報が漏洩するリスクがあります。

方法 1 — VS Code で一括検索・削除

  1. Ctrl+Shift+F で検索パネルを開く
  2. 検索: \b(print|debugPrint)\s*\(（正規表現モードをオンに）
  3. 対象フォルダを lib に限定
  4. 1 件ずつ確認して削除または置換

方法 2 — ターミナルで一覧表示

# lib/ 配下の print/debugPrint を行番号付きで表示
grep -rn "\bprint\|debugPrint" lib/ --include="*.dart"

方法 3 — logger パッケージに置き換える（推奨）

本番コードでログが必要な場合は、リリースビルドで自動的に無効化されるロガーを使います。

# pubspec.yaml に追加
dependencies:
  logger: ^2.0.0
import 'package:logger/logger.dart';

final log = Logger();

// print('message') の代わりに:
log.d('デバッグメッセージ');   // DEBUG
log.i('情報メッセージ');       // INFO
log.w('警告メッセージ');       // WARNING
log.e('エラーメッセージ');     // ERROR

Logger() はデフォルトでリリースビルドでは出力されません。

方法 4 — kReleaseMode で条件付き出力

import 'package:flutter/foundation.dart';

// リリースビルドでは実行されない
if (kDebugMode) {
  print('debug only');
}
ヒント: Pre-flight チェックの詳細欄にファイルと行番号が表示されています。 そこを参考に直接ジャンプして修正するのが最速です。
