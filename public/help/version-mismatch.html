バージョン不一致の修正

⚠️ 警告

問題

pubspec.yaml、android/app/build.gradle、ios/Runner/Info.plist の間でバージョン番号が一致していません。ストアへの提出時にリジェクトされる原因になります。

推奨: pubspec.yaml で一元管理する（Flutter 標準構成）

Flutter の標準的なセットアップでは、Android と iOS のバージョンを pubspec.yaml から自動的に引き継ぎます。

pubspec.yaml — ここだけ変更すれば OK:

version: 1.2.0+5
#         ^^^^^ ^^
#         versionName  versionCode

android/app/build.gradle — Flutter 経由に設定:

android {
    defaultConfig {
        versionCode flutter.versionCode
        versionName flutter.versionName
    }
}

ios/Runner/Info.plist — Flutter 経由に設定:

<key>CFBundleVersion</key>
<string>$(FLUTTER_BUILD_NUMBER)</string>
<key>CFBundleShortVersionString</key>
<string>$(FLUTTER_BUILD_NAME)</string>

手動で合わせる場合

各ファイルで同じバージョン番号を設定します。

  • pubspec.yaml: version: X.Y.Z+N
  • build.gradle: versionCode N / versionName "X.Y.Z"
  • Info.plist: CFBundleVersion = N, CFBundleShortVersionString = X.Y.Z
ヒント: Flutter 標準構成（flutter.versionCode / $(FLUTTER_BUILD_NUMBER)） にしておくと、pubspec.yaml の 1 か所だけ変更すれば全プラットフォームに反映されます。
