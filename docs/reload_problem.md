# 再読み込みでカードが空になる問題

## 症状

- プロセス / ログ画面で、実行中カードや履歴カードが突然空になることがある
- `npm run dev` や `node --watch` でサーバーを起動していると発生しやすい

## 原因

### 1. サーバー再起動でメモリが消える

`server/processManager.js` は、実行中プロセスの状態を `Map` にメモリ保持している。
そのため、サーバーファイルを編集して `node --watch` が再起動すると、実行中一覧・履歴・接続情報が失われる。

### 2. クライアントの自動更新が空の状態を拾う

`public/app.js` の自動更新は 10 秒ごとに一覧を再取得する。
サーバー再起動直後に空レスポンスを受けると、既存カードを空表示に上書きしてしまう。

## 再現条件

1. `npm run dev` でサーバー起動
2. プロセス / ログ画面を開く
3. サーバー側ファイルを編集して `node --watch` が再起動
4. 数秒後の自動更新でカードが消える

## 対策案

### A. UI 側で「再取得中に消さない」

- 空レスポンスを受けても、即座にカードを消さない
- 「確認中」「再接続中」の表示に寄せる
- 実装コストは低い
- ただし再起動で本当に消えたプロセスを復元するわけではない

### B. メタ情報を永続化する

- `label`, `cmd`, `cwd`, `startedAt` などを JSON ファイルに保存する
- 再起動後もカードの見た目を復元しやすくなる
- ただし PID や接続先は生きていない可能性があるので、running 状態の完全復元はできない

### C. 常駐マネージャーを分離する

- `node --watch` の影響を受けない別プロセスに管理を移す
- 最も堅牢
- その分、実装と運用は重くなる

## 結論

- 「空になる」だけを防ぐなら UI 側の対策で十分
- 「再起動後もカードを残したい」ならメタ情報の永続化が必要
- 「実行中状態まで維持したい」なら常駐マネージャー分離が必要
